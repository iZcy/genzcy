🧩 The 12 Concepts Reference
1Client-Server(W1)Know what lives where. "Validate on frontend, submit to backend."
2HTTP Lifecycle(W1)URL→DNS→server→response. Debug "why is this blank?"
3Frontend vs Backend(W2/W3)Specify which side a feature belongs to.
4Data Modeling(W3)THE most important. "User has many Orders" → right schema.
5CRUD(W3)90% of features = Create, Read, Update, Delete.
6API (REST)(W3)"GET /orders" = how frontend talks to backend.
7Auth & Authz(W3)"Login with Google" vs "Admin can delete" = different architecture.
8State Mgmt(W2/W3)"After saving, update list without refresh."
9Error Handling(W1-W4)"What if email invalid?" + "What if DB down?"
10Env & Secrets(W3/W4)API keys in .env, not code. Essential security.
11Deployment Envs(W4)Local vs staging vs prod. "Don't break my live site."
12Monitoring(W4)Logs, metrics, alerts. Know when something's wrong.
🔍 5-Level Troubleshooting Checklist
L1 — User says: "The booking form isn't working"
L2 — Browser Console: Open DevTools → any errors? Network tab → status codes?
L3 — Server Logs: Check logs → find failed request → read stack trace
L4 — Database: Open DB browser → does the table exist? Is data there?
L5 — External API: Is WhatsApp/payment working? API key valid? Rate limited?
