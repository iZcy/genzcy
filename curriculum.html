1. Program Philosophy & the 3 Goals

🎯 Goal #1 — Community

Every graduate joins a network of business-owner builders. Shared vocabulary = shared identity = natural referrals and collaboration.

🧠 Goal #2 — Common Knowledge

After 4 weeks, you speak the full-stack map fluently. Every conversation about your website becomes faster, clearer, cheaper.

📈 Goal #3 — Scale

When your business grows beyond vibe coding, you already have a trusted partner who knows your architecture. Growth returns to mentor for managed IT.

📐 Philosophy

Traditional education teaches you to write code. This teaches you to architect systems and direct AI to build them. You become an architect who can commission, verify, and maintain software.

4. The 12 Architectural Concepts (for Better Prompting)

1 Client-Server (W1)Know what lives where. "Validate on frontend, submit to backend."
2 HTTP & Request Lifecycle (W1)URL → DNS → server → response. Debug "why is this blank?"
3 Frontend vs Backend (W2/W3)Specify WHICH side a feature belongs to in prompts.
4 Data Modeling (W3)THE most important skill. "User has many Orders" → AI builds right schema.
5 CRUD Operations (W3)90% of features = Create, Read, Update, Delete. Frame requests in these terms.
6 API (REST) (W3)"GET /orders returns order history" — how frontend talks to backend.
7 Auth & Authorization (W3)"Users log in with Google" vs "Admins can delete" = different architecture.
8 State Management (W2/W3)"After saving, update list without refresh" requires state-awareness.
9 Error Handling (W1-W4)"What if email invalid?" + "What if DB down?" = robust prompts.
10 Environment & Secrets (W3/W4)API keys / DB URLs belong in .env, not code. Essential security.
11 Deployment Envs (W4)Local vs staging vs production. "Don't break my live site."
12 Monitoring & Observability (W4)Logs, metrics, alerts. Know when something's wrong, describe it.

10. The 5-Layer Diagnostic Flow

When something breaks, trace it from the user's screen to the database. By the time you call your mentor, say: "Booking form returns 500. Server log says WhatsApp API returned 401. Checked .env, API key looks correct."

Layer 1: User Report
"The booking form isn't working"
↓ What exactly happens? Error message? Blank page?
Layer 2: Browser Console
DevTools → Console tab (errors?) + Network tab (status codes?)
↓ 404 (not found) / 500 (server error) / 4xx (client)?
Layer 3: Server Logs
Check server console/log · find failed request · read stack trace
↓ Which file:line? What was passed vs expected?
Layer 4: Database
Open DB browser · table exist? Columns correct? Data there?
↓ Try manual query. Null? Wrong type? Missing?
Layer 5: External API
Is WhatsApp/payment/Maps working? API key valid? Rate limit?
↓ Outage? Expired key? Config mismatch?
