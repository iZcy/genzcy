🔍 The 5-Level Troubleshooting Checklist

When something breaks, trace it from screen to database. By the time you call for help, say: "Booking form returns 500. Server log says WhatsApp API returned 401. I checked .env, the key looks right."

Level 1: What happened?
Ask the user/report. What exactly? Error message? Blank page?
Level 2: Browser tools
Open browser developer tools. Any red errors? Network tab: what status code? 404/500/401?
Level 3: Server logs
Check the server console/log. Find the failed request. Read the error — which file:line? What was passed?
Level 4: Database
Open your DB. Does the table exist? Right columns? Try a simple query to see if data is there.
Level 5: External service
Is WhatsApp/payment working? API key valid? Not expired? Rate limited?
🛠️ Your Toolkit — What You Can Do vs What You Can Read
Things you do yourself:
  • Open VS Code, edit files, run terminal commands
  • Use Git: add, commit, push, check log
  • Run OpenCode, type prompts, see it build
  • Toggle PromptAudit, review your own log
  • Deploy your site, check if it's live
  • View analytics — visitors, leads, conversions
Things you inspect and understand:
  • Browser DevTools: inspect any page, see network requests, read console errors
  • Server logs: read error messages, identify file and line number
  • Database browser: see tables and rows, run a basic query
  • Git log: read commit history, understand what changed
  • .env file: audit secrets, check nothing is missing
  • Error reports (if configured): see what broke and when
